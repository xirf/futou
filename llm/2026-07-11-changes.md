# 2026-07-11 Changes

## P0 security hardening

- Runtime installs now validate Windows path components, require SHA-256, use
  unique staging directories, and commit only complete extractions.
- Uninstall resolves repository state and requires an exact canonical runtime
  directory before recursive deletion.
- Daemon state mutations are serialized and persisted with synced temporary
  files, atomic Windows replacement, and recoverable backups.
- Catalogue schema v2 records provider provenance and pinned artifacts. Remote,
  cached, and bundled snapshots require valid Ed25519 signatures.
- The maintainer catalogue generator discovers PHP, Node.js, MariaDB, and Deno,
  merges reviewed PostgreSQL/nginx pins, and drives signed update PRs from CI.
- Python remains deferred; Apache is omitted while its HTTPS endpoint downgrades
  downloads to HTTP.
